# Network Flexibility Design under Endogenous Uncertainty

This repository contains the code and instances accompanying the paper:

**Network Flexibility Design under Endogenous Uncertainty**  
Authors: Soroush Fatemi-Anaraki, Martin Grunow, Giovanni Pantuso, Stefan Weltge

The code implements the computational framework used in the paper. 
Given a set of plants and products, it computes a flexibility design that maximizes expected profit under supply and demand uncertainties. 
The model considers the impact of the chosen flexibility design on the underlying probability distributions of supply and demand (endogenous uncertainty). 
In addition, several classes of valid inequalities are implemented to improve computational performance.

## Usage

The implementation is written in Python and requires the Gurobi solver to tackle the underlying optimization problems.

To run an experiment, set the main experiment parameters in `main.py` and then execute:

```bash
python main.py
```

The main parameters to configure are:

```python
# Determine whether supply and/or demand are endogenous
uncertainty_configurations = ["endogenous_supply", "endogenous_demand"]

# Determine which combinations of valid inequalities to test
experiment_combinations = [
    [],                             # Approach (i) -- baseline
    ["JC", "DC"],                   # Approach (ii)
    ["DFC"],                        # Approach (iii)
    ["DFC", "DFC-S", "DFC-D"],      # Approach (iv)
    ["DFC", "DFC-D"],               # Approach (v)
    ["DFC", "DFC-S"]                # Approach (vi)
]

# Select the folder containing the instances
load_path = Path("instances/section_6_1/")

# Select the range of instance indices to run
index_range = range(1, 2)

# Define the output path for experiment results
output_path = Path("results/")
```

The `uncertainty_configurations` parameter determines whether supply uncertainty, demand uncertainty, or both are endogenous. The `experiment_combinations` parameter determines which combinations of valid inequalities are activated. The `load_path` and `index_range` parameters determine which instances are loaded, while `output_path` specifies where the results are written.

For example, the setting above runs the instance with index `1` from `instances/section_6_1/` and tests the baseline together with the valid-inequality configurations used in different sections of the paper.

├── src/                  # Main source code

├── results/              # Output files generated by experiments
